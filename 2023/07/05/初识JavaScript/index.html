初识JavaScript -- 基本语法

发表于 2023-07-05 更新于 2023-07-08 本文字数： 2k 阅读时长 ≈ 7 分钟

JavaScript是Web编程语言。绝大多数网站都使用JavaScript，所有现代Web浏览器(无论是桌面、平板还是手机浏览器)都包含JavaScript解释器，这让JavaScript成为有史以来部署最广泛的编程语言。过去十年，Node.js让浏览器之外的JavaScript编程成为可能，Node的巨大成功意味着JavaScript如今也是软件开发者最常用的编程语言。

写在前面

由于之前接触过一些高级编程语言，所以这篇文章主要也是记录JavaScript相对于其他语言的不同。
和之前所学的C、Java不同的的是:

  • JavaScript是一门高级、动态、解释型编程语言。JavaScript的变量是无类型的
  • JavaScript这个名字其实和Java没啥太大关系，JavaScript通过很长时间才从一门脚本语言成长为一门健壮高效的通用语言。
  • ES6之前的语法就是依托答辩
  • JavaScript其实是不需要;的，JavaScript会自动的匹配可能的表达式。但是为了可读性，建议使用分号避免歧义

JavaScript的算术

  • JavaScript的算术遇到上溢出、下溢出或被零整除的时候不会发生错误。上溢出发生在超出JavaScript定义的值的上限时会返回Infinity和-Infinity，下溢出发生在数值操作的结果比最小可表达数值更接近0的情况下。
  • 被零除在JavaScript不是错误，只会简单的返回无穷或负无穷。不过有一个例外:0除以0是没有意义的值，这个操作的结果是一个特殊的“非数值”(NAN)
  • 负零值也有点不同寻常，它与正零值相等(即便使用JavaScript的严格相等1比较)
  • ES2020定义了一种新的数值类型BigInt2，BigInt字面量写作一串数字后跟小写字母n.
  1. 在JavaScript中’==’为普通相等比较’===’为严格相等比较，普通比较会自动转换类型在进行比较，容易闹出不符合逻辑的笑话，例如:

null与undefined

null 是一个关键词，通常用于表示某个值不存在，undefined表示的是更深层次的不存在，变量的值未初始化就是一种undefined

符号

符号是ES6新增的一种原始类型，用作非字符串的属性名，它永远不会返回相同的值，值和变量名是一一对应的。

变量和常量作用域

变量作用域是程序源代码中一个区域，在这个区域内变量有定义。通过 let 和 const 声明的变量和常量具有块作用域。这意味着它们只在 let 和 const 语句所在的代码块中有定义。JavaScript类和函数的函数体是代码块，if/else语句的语句体、while和for循环的循环体都是代码块4。

  1. 粗略的讲，如果变量或常量声明在一对花括号中，那么这对花括号就是一个代码块。

使用var的变量声明

在ES6之前的JavaScript中，声明变量的唯一方式是使用var关键字，无法声明常量。var 的语法与 let 的语法相同。
虽然 var 和 let 具有相同的语法，但它们也有重要的区别:

  • 使用 var 声明的变量不具有块作用域。这种变量的作用域仅限于包含函数的函数体，无论它们在函数中嵌套的层次有多深。
  • 如果在函数外部使用 var ，则会声明一个全局变量。但通过 var 声明的全局变量与通过 let 声明的全局变量有一个重要区别。通过 var 声明的全局变量被实现为全局对象的属性。全局对象可以通过globalThis引用。因此，如果你在函数外部写了 var = 2 ;就相当于写了globalThis.x = 2;
  • 与通过 let 声明的变量不同，使用 var 多次声明同名变量是合法的
  • var 声明的一个最不同寻常的特性就是作用域提升。在使用 var 声明变量时，该声明会被提高到包含函数的顶部。但代码初始化仍然在代码所在位置完成，只有变量的定义转移到了函数顶部。因此对使用 var 声明的变量，可以在包含函数内部的任何地方使用而不会报错。如果初始化代码尚未运行，则变量的值可能是undefined。但初始化前是可以使用变量而不报错的，这可能会成为一个bug的来源。
